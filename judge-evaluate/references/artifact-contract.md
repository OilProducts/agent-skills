# Judge Artifact Contract

The judge emits one required artifact and may emit one optional eval report.

## Required: `verdict.json`

Required top-level keys:

- `task_id` (string)
- `verdict` (string: `pass` | `reject` | `needs-human`)
- `reasons` (array of objects)
- `required_changes` (array of strings)
- `suggested_tests` (array of strings)
- `requirements_checked` (array of requirement IDs)
- `requirements_missing` (array of requirement IDs)
- `notes` (string)

`reasons` item shape:

- `check` (string)
- `details` (string)

Example:

```json
{
  "task_id": "TASK-0042",
  "verdict": "reject",
  "reasons": [
    {"check": "tests", "details": "2 failing tests in tests/test_reset.py"},
    {"check": "spec", "details": "Missing invalid-token error path"}
  ],
  "required_changes": [
    "Implement invalid token error handling",
    "Add regression test for expired token"
  ],
  "suggested_tests": [
    "pytest -q tests/test_reset.py::test_invalid_token",
    "pytest -q tests/test_reset.py::test_expired_token"
  ],
  "requirements_checked": [
    "RQ-0102"
  ],
  "requirements_missing": [
    "NFR-0001"
  ],
  "notes": "Handoff shape valid; rejection based on eval evidence."
}
```

## Optional: `eval-results.json`

Generated by `scripts/run_eval.py`.

Contains:
- command list
- status and exit code per command
- log file paths
- pass/fail summary counts
